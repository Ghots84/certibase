# Decision Log — UX CertiBase (2026-07-09)

## Contexte
Dashboard interne CertiBase (Next.js 16 + Tailwind 4), CSM + Sales + admin. Aucun système de design cohérent : chaque page réinvente ses styles inline au cas par cas. Déclencheur : la page /concurrents (tableau comparatif de prix + timeline d'alertes), jugée "basique".

## Décisions capturées

- **Stakes** : outil interne B2B (CSM/Sales CertiPlace), pas grand public, pas réglementé. [ASSUMPTION — non confirmé explicitement, dérivé du contexte projet]
- **Identité de marque** : garder l'identité actuelle (orange `#E8651E` primaire, sidebar bleu marine `#122E47`, fond clair `#F6F7F9`). Confirmé par l'utilisateur ("oui" en réponse à "gardez-vous l'identité actuelle ?").
- **Point de friction nommé** : la façon dont la fonctionnalité "Concurrents" (et le reste du dashboard) est présentée est "assez basique".
- **Direction créative** : l'utilisateur délègue — "sois assez créatif". Signal fast-path : réponses courtes, pas d'envie d'un long aller-retour. → Passage en mode rendu d'options visuelles (creative tool "design directions") plutôt que poursuite du questionnaire.
- **Mode sombre** : non tranché — pas encore posé, à couvrir dans les options visuelles ou en question de suivi.
- **Références** : non données par l'utilisateur (a répondu "sois créatif" plutôt que de citer des apps).

## Prochaine étape
Rendu de 3-4 directions visuelles distinctes sur l'écran "Espace concurrent" (tableau comparatif + timeline d'alertes) comme hero screen, en conservant orange/bleu marine comme identité de base mais en variant densité, poids typographique, registre de marque et traitement des données tabulaires.

## Findings de revue résolus (2026-07-09)
- **critical** Contraste `status-badge` archived (2.8:1) → libellé texte passé de `text-faint` à `text-muted`, seuil AA respecté.
- **high** `text-faint` sur blanc (~3.1:1, 4 composants) → documenté explicitement dans Accessibility Floor avec justification (jamais seul porteur d'info critique) plutôt que corrigé partout (changement visuel plus large, hors scope de cette itération).
- **high** Aucun flux avec Fiches en action principale → Flux 4 ajouté (Aïcha, CSM, navigation par catégorie).
- **high** `button-primary`/`button-secondary` sans ligne comportementale → ajoutée dans Component Patterns.
- **high** Assistant absent de State Patterns → ligne dédiée ajoutée (chargement/streaming/erreur).
- **medium** Carte de validation d'import sans entrée DESIGN.md → nouveau composant `import-card` (hérite `fiche-card`), lié aux deux spines.
- **medium** Règle de couleur `stat-card` sans pendant comportemental → ligne Component Patterns ajoutée, cross-référencée.
- **medium** `confidence-bar` sans ligne Component Patterns → ajoutée (statique sur les 3 emplacements).
- **medium** Notation `{components.empty-state.tinted}` incohérente → alignée sur `tinted-*` partout.
- **medium** Migration Notes trop précis en identifiants de code → qualifié comme "repères, à vérifier avant usage".
- **medium** Utilisateurs sans couverture d'état → ligne "invitation en attente / organisation vide" ajoutée (avec `[ASSUMPTION]` explicite, cas non observé en usage réel).
- **medium** Imports sans état "aucun import" → ligne ajoutée.
- **low** (Vue d'ensemble sans état d'erreur, pas de flux Utilisateurs) → traités par ajout mineur ou acceptés tels quels avec justification inline.
